Create `apps/api/src/createApiServer/consoleRoutes.ts` with 5 handler functions. Follow the exact pattern from `usageRoutes.ts`:

1. **handleConsoleProvidersRoute** (`/api/console/providers`, GET only):
   - Returns `{ providers: Provider[], defaultProvider: "codex" }`
   - Provider type: `{ id: "codex" | "opencode", name: string, status: "available" | "unavailable" | "degraded" }`
   - Return both providers with status `"available"` as the initial implementation
   - Pattern match: check pathname === `/api/console/providers`, method !== "GET" → writeMethodNotAllowed

2. **handleConsoleStateRoute** (`/api/console/state`, GET only):
   - Reads `provider` query param (default `"codex"`)
   - Returns the full console state object per CONTEXT.md <specifics> section
   - Shape: `{ provider, live: true, region: "us-east-1", mesh: {status, latencyMs}, usage: {providerLabel, providerUsagePercent, weeklyUsagePercent}, agents: Agent[], edges: [{from, to, style}] }`
   - Agent objects include all fields from PRD: id, name, status, branch, worktree, commit, tokens (used/limit/display), resources (cpuPercent/memoryUsedMb/memoryLimitMb/memoryDisplay), uptime, position (x/y), isLead
   - For initial implementation, return fixture data matching the PRD example response shapes exactly

3. **handleConsoleAgentDetailRoute** (`/api/console/agents/{agentId}`, GET only):
   - Extracts `agentId` from pathname (split by `/`, index after `agents`)
   - Reads `provider` query param (default `"codex"`)
   - Returns detailed agent object per PRD: id, name, status, branch, worktree, commit, uptime, resources, tokens, diffSummary (filesChanged, filesStaged, files[] with path/additions/deletions/staged)
   - For initial implementation, return fixture data matching the PRD example

4. **handleConsolePreferenceRoute** (`/api/console/preferences/provider`, POST only):
   - Reads JSON body with `readJsonBodyOrWriteError` from requestParsers
   - Expects `{ provider: "codex" | "opencode" }`
   - Validates provider is one of the allowed values; returns 400 with error if invalid
   - Returns `{ success: true, provider }`
   - For initial implementation, stores in a module-level variable (no persistence needed yet)

5. **handleConsoleRefreshRoute** (`/api/console/refresh`, POST only):
   - Reads JSON body with optional `provider` field
   - Returns `{ success: true, refreshedAt: <ISO timestamp> }`
   - Simply returns a success response with current timestamp for initial implementation

All handlers follow the `ApiRouteHandler` signature from routeHelpers:
